## 📋 修复概览

## ✅ 问题1：Token Overlap设计缺陷

### 原始问题
```python
# ❌ 错误实现
overlap_lines = current_lines[-self.config.overlap_tokens//10:]
```

**问题分析：**
- 按"行"做overlap，但配置是"token"
- `overlap_tokens=50` → 取最后5行（50//10）
- 导致：
  - 长行：overlap token数远超50
  - 短行：overlap token数远低于50
  - 不稳定、不可控

### 修复方案
```python
# ✅ 正确实现
if self.config.overlap_tokens > 0:
    overlap_lines = []
    overlap_token_count = 0
    
    # 从后往前累计到overlap_tokens
    for j in range(len(current_lines) - 1, -1, -1):
        line_tok = self._count_tokens(current_lines[j] + '\n')
        if overlap_token_count + line_tok <= self.config.overlap_tokens:
            overlap_lines.insert(0, current_lines[j])
            overlap_token_count += line_tok
        else:
            break
```

**修复效果：**
- ✅ 真正的token-based overlap
- ✅ 精确控制overlap大小
- ✅ 长行/短行都稳定

## ✅ 问题2：行号计算错误

### 原始问题
```python
# ❌ 错误实现
start_line = chunk.start_line + len(sub_chunks) * len(current_lines)
end_line = chunk.start_line + len(sub_chunks) * len(current_lines) + len(current_lines)
```

**问题分析：**
- 假设每个chunk行数一样（`len(current_lines)`）
- 实际上每个chunk行数是动态的
- 导致：
  - 行号漂移
  - RAG检索定位错误
  - 代码溯源失败

### 修复方案
```python
# ✅ 正确实现
current_start_line = chunk.start_line  # 初始化

# 创建子块时
sub_chunk = CodeChunk(
    start_line=current_start_line,
    end_line=current_start_line + len(current_lines) - 1,
    ...
)

# 更新下一个块的起始行（考虑overlap）
current_start_line = current_start_line + len(sub_chunk.content.split('\n')) - len(overlap_lines)
```

**修复效果：**
- ✅ 精确追踪每个chunk的实际行数
- ✅ 行号不再漂移
- ✅ RAG检索准确定位

## ✅ 问题3：Fallback策略过于保守

### 原始问题
```python
# ❌ 错误实现
if len(current_group) > 5:  # 至少5行才创建chunk
    groups.append(current_group)
```

**问题分析：**
- 按行数判断（5行）
- 小函数、配置代码、常量定义被丢弃
- 导致：
  - AI训练数据不完整
  - 配置信息丢失
  - 重要的小代码块被忽略

### 修复方案
```python
# ✅ 正确实现
# 1. 不再按行数过滤，改为token判断
if tokens >= self.config.min_tokens:
    chunks.append(chunk)
else:
    # 2. 即使小于min_tokens，如果有意义也保留
    code_stripped = code.strip()
    if code_stripped and not code_stripped.startswith('#') and len(code_stripped) > 10:
        chunk.chunk_type = "config"  # 标记为配置类代码
        chunks.append(chunk)
```

**修复效果：**
- ✅ Token-based判断，更科学
- ✅ 保留配置代码和常量
- ✅ AI训练数据更完整

## ✅ 问题4：JavaScript解析功能有限

### 原始问题
```python
# ❌ 原始实现
func_pattern = r'(?:function|const|let|var)\s+(\w+)\s*=?\s*(?:async\s*)?\([^)]*\)\s*(?:=>)?\s*\{'
```

**问题分析：**
- 只支持基础函数声明
- 不支持：
  - class
  - 箭头函数（多种形式）
  - 对象方法
  - 嵌套结构

### 修复方案
```python
# ✅ 改进实现
# 1. 支持class
class_pattern = r'class\s+(\w+)(?:\s+extends\s+\w+)?\s*\{'

# 2. 支持多种箭头函数
patterns = [
    r'function\s+(\w+)\s*\([^)]*\)\s*\{',
    r'(?:const|let|var)\s+(\w+)\s*=\s*(?:async\s*)?\([^)]*\)\s*=>\s*\{',
    r'async\s+function\s+(\w+)\s*\([^)]*\)\s*\{',
    ...
]

# 3. 提取class中的方法
for method_match in re.finditer(r'(\w+)\s*\([^)]*\)\s*\{', code):
    methods.append(method_match.group(1))
```

**修复效果：**
- ✅ 支持class和方法提取
- ✅ 支持多种箭头函数形式
- ⚠️  仍基于正则（复杂嵌套可能失败）

**未来改进：**
- 需要集成tree-sitter（真正的AST解析）

## ⚠️ 问题5：语义边界保护（待改进）

### 当前状态
```python
# 当前实现：按行切分
for i, line in enumerate(lines):
    line_tokens = self._count_tokens(line + '\n')
    if current_tokens + line_tokens > max_tokens:
        # 直接切分
        create_chunk(current_lines)
```

**问题分析：**
- 可能切断：
  - `if` 语句（条件和body分离）
  - `for` 循环（循环头和body分离）
  - `try-except` 块（异常处理分离）

  - 代码语义不完整

def _split_by_ast_nodes(self, chunk):

### 2. ✅ 修复tiktoken异常处理
**问题：** 使用裸`except:`会捕获所有异常，包括`KeyboardInterrupt`。

```python
# ❌ 之前
try:
    self.tokenizer = tiktoken.get_encoding("cl100k_base")
except:  # 危险！
    self.tokenizer = None
```

**修复：** 使用`except Exception`，更安全。

```python
# ✅ 现在
try:
    self.tokenizer = tiktoken.get_encoding("cl100k_base")
except Exception as e:
    self.tokenizer = None
    logger.warning(f"tiktoken未安装或加载失败: {e}，使用字符估算")
```

### 3. ✅ 优化_calculate_importance，避免O(n²)
**问题：** `_enrich_context`已经计算过`dependents`，但`_calculate_importance`又重新调用`_extract_dependents`，造成重复计算。

**修复：** 复用`context`中已有的`dependents`。

```python
# ✅ 现在
if chunk.context and 'dependents' in chunk.context:
    dependents_count = len(chunk.context['dependents'])
else:
    # 向后兼容：如果context还没有，才重新计算
    dependents_count = len(self._extract_dependents(chunk, all_chunks))
```

**效果：** 性能提升，避免O(n²)复杂度。

---

### 4. ✅ 深度切分块添加函数签名上下文
**问题：** `_split_large_function`把控制流块抽成独立chunk，但没有带上函数签名，RAG检索时模型不知道在哪个函数里。

**修复：** 在content开头添加注释header，metadata中保存签名。

```python
# ✅ 现在
header_comment = f"# 来自函数: {func_signature.strip()}\n"
block_content_with_header = header_comment + '\n'.join(block_lines)

chunk.metadata = {
    'parent_function': func_name,
    'parent_signature': func_signature.strip(),  # 保存签名
    'control_type': 'If',
    'deep_split': True
}
```

**示例输出：**
```python
# 来自函数: def process_data(self, data):
if data is None:
    raise ValueError("Data cannot be None")
```

**效果：** RAG检索到控制流块时，能看到它属于哪个函数。

### 6. ✅ 文件编码错误处理
**问题：** 直接`open(..., encoding='utf-8')`，遇到GBK文件会崩溃。

**修复：** 添加编码fallback。

```python
# ✅ 现在
try:
    with open(file_path, 'r', encoding='utf-8') as f:
        content = f.read()
except UnicodeDecodeError:
    logger.warning(f"UTF-8解码失败，尝试GBK: {file_path.name}")
    try:
        with open(file_path, 'r', encoding='gbk') as f:
            content = f.read()
    except Exception as e:
        logger.error(f"文件编码错误，跳过: {file_path.name} - {e}")
        continue
```

**效果：** 支持GBK编码文件，不会崩溃。

---

### 7. ✅ 修复line_map内存泄漏
**问题：** `self.line_map`会累积所有处理过的文件内容，大型代码库会吃掉大量内存。

**修复：** 处理完一个文件就清理。

```python
# ✅ 现在
def chunk(self, ...):
    # 建立映射
    self.line_map[str(file_path)] = lines
    
    # ... 处理 ...
    
    # 处理完后清理
    if str(file_path) in self.line_map:
        del self.line_map[str(file_path)]
```

**效果：** 内存占用稳定，不会随文件数增长。

---

### 8. ✅ 添加chunk去重机制
**问题：** 同一个函数可能被`_structure_chunk`和`fallback`同时覆盖，虽然`covered_lines`有规避，但基于content hash的去重是最后保险。

**修复：** 在PostProcessor中添加去重步骤。

```python
def _deduplicate_chunks(self, chunks):
    seen_ids = set()
    unique_chunks = []
    
    for chunk in chunks:
        if chunk.id not in seen_ids:
            seen_ids.add(chunk.id)
            unique_chunks.append(chunk)
        else:
            logger.debug(f"移除重复chunk: {chunk.name}")
    
    return unique_chunks
```

**效果：** 确保没有重复chunk。

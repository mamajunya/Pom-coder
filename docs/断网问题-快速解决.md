# 断网问题快速解决方案（V3.6.2强化版）

## ❌ 错误信息

```
MaxRetryError("HTTPSConnectionPool(host='huggingface.co', port=443): 
Max retries exceeded with url: /sentence-transformers/all-MiniLM-L6-v2/resolve/main/modules.json
```

## ✅ 快速解决（3步）

### 步骤1：测试离线模式

```bash
python test_offline_mode.py
```

如果显示"✓ 离线模式测试通过"，说明模型已缓存，可以直接使用。

如果显示"✗ 模型加载失败"，继续下一步。

---

### 步骤2：联网环境下载模型

在**有网络**的电脑上运行：

```python
# 下载模型到本地
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('sentence-transformers/all-MiniLM-L6-v2')
print("✓ 模型已下载")
```

或使用命令行：

```bash
python -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('sentence-transformers/all-MiniLM-L6-v2')"
```

---

### 步骤3：使用离线启动脚本

**Windows**:
```bash
start_offline.bat
```

**Linux/Mac**:
```bash
chmod +x start_offline.sh
./start_offline.sh
```

**Python**:
```bash
python start_offline.py
```

## 🎯 验证成功

启动服务后，看到以下日志说明成功：

```
✓ Embedding模型加载完成（使用本地缓存）
```

如果看到：

```
✓ Embedding模型加载完成（在线下载）
```

说明仍在尝试联网，请检查缓存目录是否正确。

## 🔧 故障排查

### 问题1：仍然尝试联网

**检查缓存目录**：

```bash
# Windows
dir "C:\Users\你的用户名\.cache\huggingface\hub\" | findstr "all-MiniLM"

# Linux/Mac
ls ~/.cache/huggingface/hub/ | grep all-MiniLM
```

如果没有输出，说明模型未缓存。

### 问题2：权限错误

**Windows**:
```powershell
# 给予完全控制权限
icacls "C:\Users\你的用户名\.cache\huggingface" /grant 你的用户名:F /t
```

**Linux/Mac**:
```bash
# 修改权限
chmod -R 755 ~/.cache/huggingface/
```

### 问题3：路径不对

确认缓存目录结构：

```
.cache/
└── huggingface/
    └── hub/
        └── models--sentence-transformers--all-MiniLM-L6-v2/
            ├── snapshots/
            │   └── [hash]/
            │       ├── config.json
            │       ├── pytorch_model.bin
            │       └── ...
            └── refs/
```

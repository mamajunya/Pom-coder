# 知识库管理快速指南

## 🚀 快速开始

### 第一次构建知识库

```bash
# 1. 生成代码切片
python code_slicer.py --input ./src --output code_slices.json

# 2. 启动服务
python app_full.py

# 3. 打开浏览器
http://localhost:58761

# 4. 在Web界面操作
- 切换到"知识库管理"标签
- 上传 code_slices.json
- 选择"覆盖模式"
- 点击"构建知识库"
- 等待完成（自动重载）✅
```

## 📋 常用操作

### 1️⃣ 更新知识库（叠加模式）

**场景**：添加新代码到现有知识库

```
1. 生成新代码的切片
   python code_slicer.py --input ./new_module --output new_slices.json

2. 在Web界面上传 new_slices.json

3. 选择"叠加模式"（保留原有数据）

4. 点击"构建知识库"

5. 等待完成 → 自动重载 ✅
```

**优点**：
- ✅ 保留原有知识库
- ✅ 自动去重
- ✅ 只为新片段生成embedding（节省时间）

---

### 2️⃣ 重建知识库（覆盖模式）

**场景**：完全替换现有知识库

```
1. 生成新的代码切片
   python code_slicer.py --input ./src --output code_slices.json

2. 在Web界面上传 code_slices.json

3. 选择"覆盖模式"（清空原有数据）

4. 点击"构建知识库"

5. 等待完成 → 自动重载 ✅
```

**优点**：
- ✅ 完全重建，数据干净
- ✅ 适合大规模代码变更

---

### 3️⃣ 清除知识库

**场景**：删除所有知识库数据

```
1. 在Web界面点击"🗑️ 清除知识库"

2. 确认删除

3. 自动删除所有文件 ✅

4. 切换到无知识库模式 ✅
```

**删除的文件**：
- snippets.json
- embeddings.npy
- faiss_index.bin
- bm25_index.pkl
- config.json
- ov_model/

**注意**：此操作不可恢复！

---

### 4️⃣ 切换项目

**场景**：在不同项目之间切换

```
# 方法1：清除后重建
1. 点击"🗑️ 清除知识库"
2. 上传新项目的代码切片
3. 构建知识库

# 方法2：覆盖模式
1. 直接上传新项目的代码切片
2. 选择"覆盖模式"
3. 构建知识库
```

## ⚡ 自动重载功能

### 什么是自动重载？

训练完embedding后，系统会自动重新加载知识库，**无需手动重启服务**。

### 重载流程

```
构建知识库
    ↓
生成embedding
    ↓
保存文件
    ↓
自动重载 ✅
    ↓
立即可用
```

### 状态提示

- ✅ **成功**：`构建完成（叠加/覆盖模式），知识库已自动重载`
- ⚠️ **警告**：`构建完成，但重载失败，请手动重启服务`
- ❌ **失败**：显示具体错误信息

### 重载时间

- 小型知识库（<1000片段）：~1秒
- 中型知识库（1000-5000片段）：~3秒
- 大型知识库（>5000片段）：~5秒

## 🔧 故障排查

### 问题1：构建失败

**症状**：进度条卡住或显示错误

**检查**：
```bash
# 1. 检查JSON格式
python -m json.tool code_slices.json > /dev/null

# 2. 检查文件大小
ls -lh code_slices.json

# 3. 查看服务器日志
# 日志会显示具体错误信息
```

**常见原因**：
- JSON格式错误
- 文件过大（>100MB）
- 内存不足
- 依赖库未安装

### 问题2：重载失败

**症状**：构建完成但提示"重载失败"

**解决**：
```bash
# 手动重启服务
python app_full.py
```

**预防**：
- 确保有足够内存
- 检查知识库文件完整性

### 问题3：清除不完整

**症状**：部分文件未删除

**解决**：
```bash
# 手动删除
rm -rf ./knowledge_base/*

# 或者
python -c "import shutil; shutil.rmtree('./knowledge_base', ignore_errors=True)"
```

### 问题4：知识库不生效

**症状**：构建完成但代码生成未使用知识库

**检查**：
```bash
# 1. 检查知识库文件
ls -lh ./knowledge_base/

# 2. 检查系统信息API
curl http://localhost:58761/api/system/info

# 3. 查看 knowledge_base_loaded 字段
```

**解决**：
- 重启服务
- 重新构建知识库

## 📊 性能优化

### 1. 批量处理

**推荐**：一次性处理多个文件

```bash
# 不推荐：多次小批量
python code_slicer.py --input ./module1 --output slices1.json
python code_slicer.py --input ./module2 --output slices2.json
# 然后分别上传

# 推荐：一次性处理
python code_slicer.py --input ./src --output code_slices.json
# 一次上传
```

### 2. 控制片段数量

**推荐**：保持知识库在合理大小

```bash
# 使用更大的max_tokens减少片段数
python code_slicer.py \
  --input ./src \
  --output code_slices.json \
  --max-tokens 800  # 默认512

# 或者过滤低质量片段
# V3.5会自动过滤质量评分<5的片段
```

### 3. 使用NPU加速

**推荐**：如果有Intel Core Ultra处理器

```
在Web界面：
- 勾选"使用NPU加速"
- 速度提升约3倍
```

## ❓ 常见问题

**Q: 叠加模式会重复吗？**  
A: 不会，系统会自动去重（基于代码内容哈希）

**Q: 覆盖模式会删除文件吗？**  
A: 会删除旧的知识库文件，但不会删除原始代码切片JSON

**Q: 自动重载失败怎么办？**  
A: 手动重启服务即可：`python app_full.py`

**Q: 清除操作可以恢复吗？**  
A: 不可以，建议清除前备份重要文件

**Q: 知识库多大合适？**  
A: 建议1000-5000个片段，太少效果不好，太多影响性能

**Q: 需要重启服务吗？**  
A: V3.6版本后不需要，自动重载功能会自动刷新知识库

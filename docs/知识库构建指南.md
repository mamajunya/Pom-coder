# 知识库构建指南

## 📋 概述

本指南介绍如何使用V3.5代码切片器和知识库构建工具创建高质量的代码知识库。

## 🔄 完整流程

### 步骤1：代码切片

使用V3.5 S+级代码切片器将代码库切分为语义完整的代码块：

```bash
# 基础用法
python code_slicer.py --input ./src --output code_slices.json

# 高级选项
python code_slicer.py \
  --input ./src \
  --output code_slices.json \
  --max-tokens 500 \
  --overlap-tokens 50 \
  --languages python,javascript
```

**输出格式**（V3.5）：
```json
[
  {
    "id": "abc123...",
    "content": "def my_function():\n    ...",
    "file_path": "src/module.py",
    "language": "python",
    "tokens": 150,
    "start_line": 42,
    "end_line": 45,
    "line_indices": [42, 43, 44, 45],
    "chunk_type": "function",
    "name": "my_function",
    "summary": "执行某项功能的函数",
    "imports": ["import os", "from typing import List"],
    "context": {
      "parent_class": "MyClass",
      "file_summary": "模块功能说明",
      "neighbors": [...]
    },
    "metadata": {
      "complexity": 5
    },
    "quality_score": 9.0,
    "semantic_completeness": 1.0
  }
]
```

### 步骤2：构建知识库

使用`build_knowledge_base_npu.py`将代码块转换为向量索引：

```bash
# 基础用法（CPU）
python build_knowledge_base_npu.py \
  --source code_slices.json \
  --output ./knowledge_base

# 使用Intel NPU加速
python build_knowledge_base_npu.py \
  --source code_slices.json \
  --output ./knowledge_base \
  --model sentence-transformers/all-MiniLM-L6-v2

# 禁用NPU（强制使用CPU）
python build_knowledge_base_npu.py \
  --source code_slices.json \
  --output ./knowledge_base \
  --no-npu
```

### 步骤3：验证知识库

检查生成的文件：

```bash
knowledge_base/
├── config.json           # 配置信息
├── snippets.json         # 代码片段（保留V3.5所有字段）
├── embeddings.npy        # 向量表示
├── faiss_index.bin       # FAISS向量索引
├── bm25_index.pkl        # BM25关键词索引
└── ov_model/             # OpenVINO模型（如果使用NPU）
    ├── openvino_model.xml
    ├── openvino_model.bin
    └── ...
```

## 🔧 高级功能

### 增量更新（叠加模式）

默认情况下，知识库构建器使用叠加模式，会自动合并现有知识库和新代码块：

```bash
# 第一次构建
python build_knowledge_base_npu.py --source code_slices_v1.json --output ./kb

# 增量更新（自动去重）
python build_knowledge_base_npu.py --source code_slices_v2.json --output ./kb
```

**特性**：
- ✅ 自动去重（基于代码内容哈希）
- ✅ 只为新片段生成embedding（节省时间）
- ✅ 保留现有片段的所有信息

### 覆盖模式

如果需要完全重建知识库，修改代码启用覆盖模式：

```python
builder = NPUKnowledgeBaseBuilder(
    embedding_model="...",
    output_dir="./knowledge_base",
    use_npu=False,
    append_mode=False  # 覆盖模式
)
```

### 从目录直接构建

也可以跳过切片步骤，直接从代码目录构建：

```bash
python build_knowledge_base_npu.py \
  --source ./src \
  --output ./knowledge_base \
  --extensions .py,.js,.java \
  --max-files 1000
```

**注意**：这种方式不会使用V3.5的高级特性（质量评分、语义完整性等）。

## 📊 V3.5增强特性

### 质量评分

V3.5的`quality_score`字段会保存到metadata，可用于：

```python
# 过滤低质量代码块
high_quality_snippets = [
    s for s in snippets 
    if s.metadata.get('quality_score', 0) >= 8.0
]

# 加权检索（高质量代码块权重更高）
scores = base_scores * quality_weights
```

### 语义完整性

`semantic_completeness`字段表示代码块是否完整：

```python
# 只使用完整的代码块
complete_snippets = [
    s for s in snippets 
    if s.metadata.get('semantic_completeness', 1.0) >= 0.9
]
```

### 精确定位

使用`line_indices`字段精确定位代码：

```python
# 获取代码块的精确行号
start_line = snippet.metadata['start_line']
end_line = snippet.metadata['end_line']
line_indices = snippet.metadata.get('line_indices', [])

# 在IDE中跳转到代码位置
open_in_editor(snippet.path, start_line)
```

### 上下文增强

`context`字段提供丰富的上下文信息：

```python
context = snippet.metadata.get('context', {})

# 获取父类信息
parent_class = context.get('parent_class')

# 获取文件摘要
file_summary = context.get('file_summary')

# 获取相邻代码块
neighbors = context.get('neighbors', [])
```

## 🔍 检索示例

### 基础检索

```python
from src.rag_code_generator.retrieval import HybridRetriever

retriever = HybridRetriever(
    knowledge_base_dir="./knowledge_base"
)

# 检索相关代码
results = retriever.retrieve(
    query="如何处理文件上传",
    top_k=5
)

for result in results:
    print(f"文件: {result['path']}")
    print(f"代码: {result['code']}")
    print(f"质量: {result.get('metadata', {}).get('quality_score', 'N/A')}")
    print("-" * 60)
```

### 高级检索（利用V3.5特性）

```python
# 只检索高质量、完整的代码块
results = retriever.retrieve(
    query="如何处理文件上传",
    top_k=10,
    filters={
        'quality_score': {'$gte': 8.0},
        'semantic_completeness': {'$gte': 0.9}
    }
)

# 按质量评分排序
results = sorted(
    results, 
    key=lambda x: x.get('metadata', {}).get('quality_score', 0),
    reverse=True
)
```

## ⚠️ 常见问题

### Q1: 构建知识库时报错 `KeyError: 'code'`

**原因**：使用了旧版本的`build_knowledge_base_npu.py`，不兼容V3.5格式。

**解决**：更新到V3.5.2或更高版本，已修复兼容性问题。

### Q2: NPU加速不可用

**检查**：
```bash
python -c "import openvino as ov; print(ov.Core().available_devices)"
```

如果没有`NPU`设备，使用`--no-npu`强制CPU模式。

### Q3: 内存不足

**解决方案**：
- 减少`--max-files`参数
- 增加`--max-tokens`参数（生成更少的代码块）
- 分批构建，使用叠加模式合并

### Q4: 生成的embedding维度不匹配

**原因**：更换了embedding模型。

**解决**：删除现有知识库，重新构建：
```bash
rm -rf ./knowledge_base
python build_knowledge_base_npu.py --source code_slices.json --output ./knowledge_base
```

## 📈 性能优化

### 批处理大小

根据可用内存调整批处理大小：

```python
builder.generate_embeddings(batch_size=32)  # 默认
builder.generate_embeddings(batch_size=64)  # 更快，需要更多内存
builder.generate_embeddings(batch_size=16)  # 更慢，节省内存
```

### 模型选择

不同模型的权衡：

| 模型 | 维度 | 速度 | 质量 | 推荐场景 |
|------|------|------|------|----------|
| all-MiniLM-L6-v2 | 384 | 快 | 中 | 快速原型 |
| all-mpnet-base-v2 | 768 | 中 | 高 | 生产环境 |
| code-search-net | 768 | 中 | 高 | 代码专用 |

### NPU加速

Intel Core Ultra处理器的NPU可以显著加速embedding生成：

- **CPU模式**：~100 samples/sec
- **NPU模式**：~300 samples/sec（3x加速）

## 🎯 最佳实践

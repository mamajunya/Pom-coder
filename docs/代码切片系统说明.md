# 精致代码切片系统使用指南

## 📋 版本历史

### V2.1 (2024) - 核心BUG修复版 ✅

**修复的关键问题：**

1. **✅ Token overlap修复**
   - 问题：之前按"行"做overlap，导致不稳定
   - 修复：改为真正的token-based overlap
   - 影响：overlap精确可控，长行/短行都稳定

2. **✅ 行号计算修复**
   - 问题：start_line/end_line计算错误，会漂移
   - 修复：精确追踪每个chunk的实际行数
   - 影响：RAG检索和代码定位准确

3. **✅ Fallback策略改进**
   - 问题：按行数判断，丢弃小代码块
   - 修复：改为token-based，保留配置代码
   - 影响：AI训练数据更完整

4. **✅ JavaScript解析改进**
   - 问题：不支持class、箭头函数
   - 修复：支持更多JS模式
   - 限制：仍基于正则（未来需tree-sitter）

5. **⚠️ 语义边界保护（待改进）**
   - 当前：按行切分，可能切断控制流
   - 未来：需AST子节点级切分

## 🎯 核心特性

### 1. 多级切片策略
- **Level 1: 结构切片** - 按函数/类划分
- **Level 2: Token切片** - 超长块再切分
- **Level 3: Fallback切片** - 处理未覆盖代码

### 2. Token级精确控制
- 最大Token数：512（可配置）
- 最小Token数：50（可配置）
- 重叠Token数：50（10% overlap）

### 3. 语义完整性保证
- 优先保持函数/类完整
- 保留import语句
- 保留注释和文档字符串

### 4. 丰富的元数据
- 代码类型（function/class/method/fragment）
- Token统计
- 复杂度分析
- 质量评分
- 依赖关系

## 🚀 快速开始

### 基础用法

```bash
python code_slicer.py --source ./src --output chunks.json
```

### 高级用法

```bash
python code_slicer.py \
  --source ./src \
  --output chunks.json \
  --max-tokens 512 \
  --min-tokens 50 \
  --overlap 50 \
  --strategy hybrid \
  --format json
```

## ⚙️ 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--source` | 必填 | 源代码目录 |
| `--output` | `code_chunks_v2.json` | 输出文件路径 |
| `--max-tokens` | 512 | 最大Token数 |
| `--min-tokens` | 50 | 最小Token数 |
| `--overlap` | 50 | 重叠Token数 |
| `--strategy` | `hybrid` | 切片策略（structure/token/hybrid） |
| `--format` | `json` | 输出格式（json/jsonl） |

## 🎨 切片策略对比

### Structure（结构优先）
- ✅ 语义完整
- ✅ 易于理解
- ❌ Token数不均匀

### Token（Token优先）
- ✅ Token数均匀
- ✅ 精确控制
- ❌ 可能断裂语义

### Hybrid（混合策略）⭐ 推荐
- ✅ 结构完整
- ✅ Token可控
- ✅ 覆盖全面
- ✅ 工业标准

## 📈 质量评分标准

质量分数（0-10分）计算规则：

- 基础分：5.0
- Token数合理（50-512）：+2.0
- 有函数/类名称：+1.0
- 有文档字符串：+1.0
- 复杂度适中（1-10）：+1.0

## 🔧 编程接口

```python
from code_slicer import CodeSlicerPipeline, SlicerConfig

# 创建配置
config = SlicerConfig(
    max_tokens=512,
    min_tokens=50,
    overlap_tokens=50,
    strategy="hybrid",
    preserve_structure=True,
    include_imports=True,
    include_comments=True
)

# 创建流水线
pipeline = CodeSlicerPipeline(config)

# 运行切片
pipeline.run(source_dir="./src", output_file="chunks.json")
```

## 🎯 最佳实践

- 推荐：10%-20% of max_tokens
- 最小：20 tokens
- 最大：100 tokens

### 3. 策略选择
- **小项目**：structure
- **大项目**：hybrid ⭐
- **特殊需求**：token

# 代码切片存储说明

## 文件存储位置

### 1. 切片结果文件

代码切片完成后，结果保存在项目根目录下：

```
项目根目录/
├── code_slices.json          # 切片结果文件（默认）
├── uploads/                  # 上传文件目录
│   ├── code_files/          # 上传的源代码文件
│   └── *.json               # 上传的知识库文件
└── knowledge_base/          # 构建的知识库
```

### 2. 详细说明

#### 切片结果文件
- **文件名**: `code_slices.json`
- **位置**: 项目根目录
- **格式**: JSON数组
- **内容**: 所有提取的代码片段（函数、类、方法等）

#### 上传的源代码文件
- **目录**: `./uploads/code_files/`
- **内容**: 用户通过Web界面上传的源代码文件
- **保留**: 上传后会保留在此目录，供后续使用

#### 知识库文件
- **目录**: `./knowledge_base/`
- **内容**: 基于切片结果构建的Embedding索引
- **文件**: 
  - `index.faiss` - FAISS索引文件
  - `metadata.json` - 元数据文件

## 切片结果文件格式

### code_slices.json 结构

```json
[
  {
    "type": "function",
    "name": "quicksort",
    "code": "def quicksort(arr):\n    if len(arr) <= 1:\n        return arr\n    ...",
    "file_path": "src/utils.py",
    "start_line": 10,
    "end_line": 25,
    "language": "python",
    "complexity": 5,
    "docstring": "快速排序算法实现",
    "metadata": {
      "parameters": ["arr"],
      "returns": "list",
      "decorators": []
    }
  },
  {
    "type": "class",
    "name": "DataProcessor",
    "code": "class DataProcessor:\n    def __init__(self):\n        ...",
    "file_path": "src/processor.py",
    "start_line": 30,
    "end_line": 100,
    "language": "python",
    "complexity": 8,
    "docstring": "数据处理器类",
    "metadata": {
      "methods": ["process", "validate", "save"],
      "base_classes": ["BaseProcessor"]
    }
  }
]
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 代码片段类型（function/class/method） |
| name | string | 函数/类名称 |
| code | string | 完整代码内容 |
| file_path | string | 源文件路径 |
| start_line | int | 起始行号 |
| end_line | int | 结束行号 |
| language | string | 编程语言（python/javascript） |
| complexity | int | 代码复杂度（1-10） |
| docstring | string | 文档字符串 |
| metadata | object | 额外元数据 |

## 使用流程

### 1. 代码切片（目录模式）

```bash
# 通过Web界面
1. 打开 http://localhost:58761
2. 进入"代码切片"标签页
3. 选择"指定目录"模式
4. 输入目录路径：./src
5. 点击"开始切片"

# 结果保存到
./code_slices.json
```

### 2. 代码切片（文件上传模式）

```bash
# 通过Web界面
1. 打开 http://localhost:58761
2. 进入"代码切片"标签页
3. 选择"上传文件"模式
4. 选择要上传的文件（支持多选）
5. 点击"开始切片"

# 上传的文件保存到
./uploads/code_files/your_file.py

# 切片结果保存到
./code_slices.json
```

### 3. 构建知识库

```bash
# 通过Web界面
1. 打开 http://localhost:58761
2. 进入"知识库管理"标签页
3. 上传 code_slices.json 文件
4. 选择Embedding模型
5. 点击"构建知识库"

# 知识库保存到
./knowledge_base/
├── index.faiss
└── metadata.json
```

### 4. 使用知识库生成代码

```bash
# 通过Web界面
1. 打开 http://localhost:58761
2. 进入"代码生成"标签页
3. 输入代码需求
4. 系统自动检索知识库并生成代码
```

## 命令行使用

### 直接使用code_slicer.py

```bash
# 切片指定目录
python code_slicer.py \
  --directory ./src \
  --extensions .py .js \
  --output my_slices.json

# 结果保存到
./my_slices.json
```

### 使用API

```bash
# 目录模式
curl http://localhost:58761/api/code-slicer/slice \
  -H "Content-Type: application/json" \
  -d '{
    "directory": "./src",
    "extensions": [".py", ".js"],
    "max_files": 1000
  }'

# 结果保存到
./code_slices.json
```

## 文件管理

### 查看切片结果

```bash
# 查看文件大小
ls -lh code_slices.json

# 查看片段数量
cat code_slices.json | jq '. | length'

# 查看前5个片段
cat code_slices.json | jq '.[0:5]'

# 按类型统计
cat code_slices.json | jq 'group_by(.type) | map({type: .[0].type, count: length})'
```

### 清理文件

```bash
# 删除切片结果
rm code_slices.json

# 删除上传的文件
rm -rf uploads/code_files/*

# 删除知识库
rm -rf knowledge_base/
```

### 备份文件

```bash
# 备份切片结果
cp code_slices.json code_slices_backup_$(date +%Y%m%d).json

# 备份知识库
tar -czf knowledge_base_backup_$(date +%Y%m%d).tar.gz knowledge_base/
```

## 存储空间估算

### 切片结果文件大小

- **小型项目**（<100个文件）: 约 1-5 MB
- **中型项目**（100-500个文件）: 约 5-20 MB
- **大型项目**（>500个文件）: 约 20-100 MB

### 知识库文件大小

- **FAISS索引**: 约为切片文件的 2-3 倍
- **元数据**: 约为切片文件的 0.5 倍
- **总计**: 约为切片文件的 2.5-3.5 倍

### 示例

```
项目规模: 200个Python文件
切片结果: code_slices.json (10 MB)
知识库:
  ├── index.faiss (25 MB)
  └── metadata.json (5 MB)
总计: 40 MB
```

## 配置选项

### 修改默认输出文件名

在 `app_full.py` 中修改：

```python
# 当前默认值
output_file = "code_slices.json"

# 修改为自定义名称
output_file = "my_custom_slices.json"
```

### 修改上传目录

```python
# 当前默认值
upload_dir = Path("./uploads/code_files")

# 修改为自定义目录
upload_dir = Path("/path/to/custom/uploads")
```

### 修改知识库目录

在构建知识库时指定：

```python
builder = NPUKnowledgeBaseBuilder(
    embedding_model="...",
    output_dir="./my_knowledge_base",  # 自定义目录
    use_npu=True
)
```

## 常见问题

### Q1: 切片文件在哪里？
**A**: 在项目根目录下的 `code_slices.json` 文件。

### Q2: 上传的文件保存在哪里？
**A**: 在 `./uploads/code_files/` 目录下。

### Q3: 知识库文件在哪里？
**A**: 在 `./knowledge_base/` 目录下。

### Q4: 如何查看切片结果？
**A**: 
```bash
# 使用文本编辑器
code code_slices.json

# 或使用jq工具
cat code_slices.json | jq '.'
```

### Q5: 切片文件会被覆盖吗？
**A**: 是的，每次切片都会覆盖 `code_slices.json`。建议在切片前备份。

### Q6: 可以同时保存多个切片结果吗？
**A**: 可以，通过命令行指定不同的输出文件名：
```bash
python code_slicer.py --directory ./src --output src_slices.json
python code_slicer.py --directory ./tests --output test_slices.json
```

### Q7: 切片文件可以手动编辑吗？
**A**: 可以，但要保持JSON格式正确。建议使用JSON编辑器。

### Q8: 如何合并多个切片文件？
**A**: 
```bash
# 使用jq合并
jq -s 'add' file1.json file2.json > merged.json
```

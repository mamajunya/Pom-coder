# 代码切片功能说明

## 功能概述

代码切片工具可以自动扫描文件夹，筛选指定后缀的代码文件，提取函数和类，生成结构化的知识库数据。

## 主要特性

### 1. 自动文件夹扫描
- 递归扫描指定目录及所有子目录
- 自动跳过常见排除目录：
  - `node_modules` - Node.js依赖
  - `__pycache__` - Python缓存
  - `.git` - Git仓库
  - `venv`, `env` - Python虚拟环境
  - `dist`, `build` - 构建输出
  - `.pytest_cache` - 测试缓存

### 2. 智能文件筛选
- 支持多种文件扩展名：
  - Python: `.py`
  - JavaScript/TypeScript: `.js`, `.ts`, `.jsx`, `.tsx`
  - Java: `.java`
  - C/C++: `.c`, `.cpp`
  - Go: `.go`
  - Rust: `.rs`
- 可自定义扩展名列表
- 限制最大文件数，避免处理时间过长

### 3. 代码提取
- **Python**：
  - 函数（包括async函数）
  - 类（包括方法、基类）
  - 提取文档字符串
  - 计算圈复杂度
  - 提取参数信息
  
- **JavaScript/TypeScript**：
  - 函数定义
  - 类和方法
  - 箭头函数

### 4. 智能过滤
- 最小代码长度：过滤太短的代码片段（默认50字符）
- 最大代码长度：过滤太长的代码片段（默认5000字符）
- 自动跳过空函数和简单getter/setter

### 5. 结构化输出
每个代码片段包含：
```json
{
  "code": "完整代码",
  "path": "文件路径",
  "language": "编程语言",
  "name": "函数/类名称",
  "type": "function/class/method",
  "description": "描述信息",
  "start_line": 1,
  "end_line": 10,
  "complexity": 5,
  "metadata": {
    "args": ["参数列表"],
    "docstring": "文档字符串",
    "methods": ["方法列表"]
  }
}
```

## 使用方法

### Web界面

1. 打开浏览器：http://localhost:58761
2. 点击"代码切片"标签页
3. 选择"扫描文件夹"模式（推荐）
4. 配置参数：
   - **源代码目录**：输入要扫描的文件夹路径
   - **文件扩展名**：选择要处理的文件类型（如`.py,.js,.ts`）
   - **最大文件数**：限制处理的文件数量
   - **代码长度范围**：过滤代码片段大小
5. 点击"开始切片"
6. 等待处理完成，查看统计信息
7. 在"知识库管理"标签页上传生成的JSON文件

### 命令行

```bash
python code_slicer.py \
  --source ./src \
  --output code_slices.json \
  --extensions .py,.js \
  --max-files 1000 \
  --min-length 50 \
  --max-length 5000
```

## 统计信息

切片完成后会显示：
- **总片段数**：提取的代码片段总数
- **处理文件数**：扫描的文件数量
- **按类型统计**：function、class、method的数量
- **按语言统计**：各语言的片段数量
- **平均复杂度**：代码的平均圈复杂度
- **平均长度**：代码片段的平均字符数

## 常见问题

### Q: 为什么某些文件没有被处理？
A: 可能原因：
1. 文件在排除目录中（如node_modules）
2. 文件扩展名不在筛选列表中
3. 达到最大文件数限制
4. 文件解析失败（语法错误）

### Q: 如何处理大型代码库？
A: 建议：
1. 分批处理不同的目录
2. 提高最小代码长度，过滤简单函数
3. 降低最大代码长度，避免超长函数
4. 限制文件扩展名，只处理主要语言

### Q: 切片后如何使用？
A: 流程：
1. 代码切片 → 生成`code_slices.json`
2. 知识库管理 → 上传JSON文件
3. 构建知识库 → 生成Embedding索引
4. AI对话 → 启用RAG检索代码

### Q: 支持哪些编程语言？
A: 当前支持：
- **完整支持**：Python（AST解析）
- **基础支持**：JavaScript/TypeScript（正则匹配）
- **计划支持**：Java, C++, Go, Rust

## 性能优化

### 提高处理速度
1. 减少文件扩展名数量
2. 提高最小代码长度
3. 降低最大文件数
4. 使用SSD存储

### 提高质量
1. 调整代码长度范围
2. 选择核心代码目录
3. 排除测试和示例代码
4. 定期更新切片数据

## 最佳实践

1. **定期更新**：代码变化后重新切片
2. **分类管理**：不同项目使用不同的切片文件
3. **质量优先**：宁少勿滥，保证代码质量
4. **增量更新**：使用叠加模式追加新代码
5. **备份数据**：保存切片JSON文件
